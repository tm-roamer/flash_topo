root权限查看示例代码

Top

因为示例代码放在demo目录下,工程跑起来需要root权限才行,有些系统没有root权限的用户,按照如下步骤添加.

在工程的根目录下有/ddl文件夹,里面有查找cupid_init.sql文件,第5行是root用户的插入语句:

INSERT INTO `auth_user` VALUES (1,'root','超级管理员','761b838fca2d2b3aa1c0c60812300cb9d81af4b0',NULL,NULL,NULL,1,NULL,0,0,0,0,0,NULL,NULL);

切记是导入本地数据库.

创建示例代码的数据库表

Top

由于示例代码库表默认是不会创建的,需要我们手动创建,过程很简单,打开TableExampleObject示例对象,里面就有建表语句.导入本地数据库即可.

示例如下:

CREATE TABLE `aa_demo_to` (
	`id` BIGINT(10) NOT NULL COMMENT '主键',
	`required` VARCHAR(30) NULL COMMENT '必填项',
	`requiredIfChecked` VARCHAR(30) NULL COMMENT '必填项check',
	`minlength` VARCHAR(200) NULL COMMENT '最小长度',
	`email` VARCHAR(100) NULL COMMENT '邮箱',
	`ipBefore` VARCHAR(35) NULL COMMENT '开始ip',
	`datetime` VARCHAR(30) NULL COMMENT '日期时间',
	`dtBefore` VARCHAR(30) NULL COMMENT '开始时间',
	`regex` VARCHAR(30) NULL COMMENT '正则',
	`pwd` VARCHAR(30) NULL COMMENT '密码',
	`number` VARCHAR(30) NULL COMMENT '数字',
	`min` INT(30) NULL COMMENT '最小数字',
	`port` INT(30) NULL COMMENT '端口',
	`gt` INT(30) NULL COMMENT '大于'
)
COMMENT='这个对象,是页面模版组件的一个演示表, 主要模拟表格的展示,增删改和验证.查询索引'
ENGINE=InnoDB
ROW_FORMAT=DEFAULT
